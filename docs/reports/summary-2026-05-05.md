# סיכום בדיקה מקיפה — RunningInWar
## תאריך: 2026-05-05

## מצב כללי — 6/10

האפליקציה בנויה על אדריכלות נקייה ומקצועית (Context, Hooks, lazy-loading, PWA, offline cache). הבעיה המרכזית: **שני הפיצ'רים הליבתיים — מסלול ריצה ואזעקות — לא עובדים בפרודקשן.** המסלול מצייר קווים ישרים דרך בתים, ומערכת האזעקות לא מחוברת לנתוני משתמש. מחוץ לזה, האפליקציה נראית טוב, נטענת מהר, ופיצ'ר ה-SOS מרשים.

---

## נושאים דחופים לטיפול 🔴

### 1. CORS — מקלטים ציבוריים ואזעקות לא עובדים בפרודקשן
**מקור: קוד + דפדפן**

שתי קריאות API נחסמות ב-CORS בדפדפן:
- `data.gov.il` — מקלטים ציבוריים לא נטענים בכלל. כל בחירת עיר → alert שגיאה.
- `/oref/WarningMessages/alert/alerts.json` — ה-proxy עובד רק ב-dev server (Vite). בפרודקשן (Vercel) — 404. האפליקציה מייצרת **100+ שגיאות console בדקה** בגלל polling כושל.

**תיקון:** הוסף Vercel Serverless Functions:
- `/api/shelters?city=...` → proxy ל-data.gov.il
- `/api/alerts` → proxy ל-OREF

---

### 2. מסלול הריצה — קווים ישרים, לא רחובות
**מקור: קוד + דפדפן**

`routeAlgorithm.js` בורר waypoints חכם (מקלטים בקרבה, nearest-neighbour) — אבל `RoutePolyline` מחבר אותם בקו ישר. **אין קריאה לאף Routing API.** המסלול עובר דרך בתים, חצרות, כבישים מהירים — לא ניתן לרוץ בפועל.

בדיקת הדפדפן: קו המסלול **לא הופיע כלל** על המפה לאחר החישוב (ייתכן שקשור ל-async).

**תיקון:** שלב OSRM (`router.project-osrm.org/route/v1/foot/`) לכל segment בין waypoints. ראה קוד מפורט בדוח הקוד.

---

### 3. האזעקה לעולם לא מופעלת
**מקור: קוד**

`useAlerts(userCity)` נקרא עם `null` בכל הדפים — ה-hook מכיל `if (!userCity) return` ולא עושה כלום. גם אם ה-CORS יתוקן, המשתמש לא יקבל אזעקות כי `userCity` לא מועבר.

**תיקון:** העבר את עיר המשתמש מהפרופיל (פונקציית `saveUserCity` קיימת ב-firestore.js אבל לא נקראת).

## נושאים חשובים 🟡

### 5. Firebase — session נמחק בסגירת טאב
**מקור: קוד**
`browserSessionPersistence` ← צריך `browserLocalPersistence` ל-PWA.

### 6. Race condition בין cache ל-Firestore
**מקור: קוד**
Cache יכול לדרוס נתונים חדשים מ-Firestore. מקלטים ישנים שנמחקו עלולים להופיע.

### 7. SOS מציג 86 ק"מ כשאין GPS
**מקור: דפדפן**
הפול-אאוט של `useLocation` שם את המשתמש בתל אביב — SOS מחשב מרחק מתל אביב. מסוכן בחירום.

### 8. PWA icon 404
**מקור: דפדפן**
`/icons/icon-192.png` לא קיים → "הוסף למסך הבית" לא עובד כראוי.

### 9. ציון בטיחות תמיד 0%
**מקור: דפדפן**
A→B mode מציג 0% ללא הסבר. כנראה קשור לאי-הופעת קו המסלול.

### 10. SOSOverlay — flooding ל-Nominatim
**מקור: קוד**
`useEffect([shelter])` ← צריך `useEffect([shelter.id])` כדי לא לשלוח reverse-geocode על כל render.

## בעיה מרכזית: אלגוריתם המסלול

**מה קורה היום:** האלגוריתם בוחר waypoints חכמים (מקלטים בקרבה למסלול, nearest-neighbour) אבל מציג אותם כקווים ישרים. המשתמש רואה מסלול שלא ניתן לרוץ אותו.

**מה צריך:** שילוב OSRM לכל segment בין waypoints:
```js
async function streetRouteBetween(a, b) {
  const url = `https://router.project-osrm.org/route/v1/foot/${a.lng},${a.lat};${b.lng},${b.lat}?overview=full&geometries=geojson`
  const res = await fetch(url)
  const data = await res.json()
  return data.routes[0].geometry.coordinates.map(([lng, lat]) => ({ lat, lng }))
}
```

**עדיפות:** גבוהה — זה הפיצ'ר המרכזי של האפליקציה.
