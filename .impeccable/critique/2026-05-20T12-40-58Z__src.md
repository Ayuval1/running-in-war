## Overall Impression

Core loop is solid. "Urban Vigil" concept has real teeth in map and emergency flow. Surface-level inconsistency — token drift, banned patterns, profile page abandoning the design language — drag it down. Fix four things and this app punches above its weight.

## What's Working

1. **SOS Overlay** — Full-screen red takeover with live bearing arrow, distance, elapsed timer. Wake lock, vibrate, Waze/Maps deeplinks, "next shelter" fallback. This is the product's core strength.
2. **Placement Mode UX** — Two-tab "pin vs. address" switcher for adding shelters. Smart progressive disclosure.
3. **GPS Indicator** — Animated ping dot in top bar is textbook use of "glow = live signal" principle.

## Priority Issues

**[P1] Side-stripe border on ShelterCard**
- src/pages/HomePage.jsx:57 — borderRight: '3px solid ${type.color}'
- Why: Explicitly banned by DESIGN.md. #1 AI slop tell in list items.
- Fix: Replace with colored dot or shelter-type icon as leading element. Use type.color as bearing arrow column background tint instead.

**[P1] alert() and confirm() throughout the codebase**
- HomePage.jsx:119, MapPage.jsx:52,57,73,454,481,482
- Why: confirm('למחוק?') renders OK/Cancel in OS language (often English) inside a Hebrew app. Breaks design trust in stressful moment.
- Fix: Inline "are you sure?" state inside cards. Toast-style inline feedback for errors.

**[P1] Profile page is a SaaS card grid**
- src/pages/ProfilePage.jsx:87-245 — Five identical bg-brand-card border rounded-2xl p-5 blocks
- Why: Exact "identical card grids" anti-pattern from DESIGN.md. Abandons the design language.
- Fix: Collapse home-location card. Group sharing/install/language as row items under "הגדרות" section.

**[P1] tactical-grid background on HomePage**
- src/pages/HomePage.jsx:141
- Why: DESIGN.md explicitly bans military/tactical visual language.
- Fix: Remove tactical-grid class. The gradient is sufficient.

**[P2] Design token drift**
- text-white/40 → text-muted (#3D7070)
- bg-green-*/text-green-* → brand-neon variants
- bg-yellow-600 → data-amber (#FFB800)

## Persona Red Flags

**Casey (Distracted Mobile User)**: App opens with disabled SOS if no shelters added. Runner under stress sees dead end. No persistence guarantee if tabs out during SOS.

**Jordan (First-Timer)**: No guided first-run flow. Must discover map, understand pin vs address mode, navigate back — all without help. Errors say "שגיאה בשמירה" with no recovery path.

**Omer (Dedicated Israeli Runner)**: Home page is setup dashboard, not pre-run check. No "I'm going for a run" simplified mode. Readiness bar visible every session even after setup is done.
